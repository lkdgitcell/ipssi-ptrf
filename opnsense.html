Pare-feu OPNsense

Sécurisation et filtrage du réseau de la Maison des Ligues de Lorraine (M2L)

BTS SIO SISR — E6 M2L Seul 01/09/2025 - 31/08/2026

Contexte

La Maison des Ligues de Lorraine (M2L) est un établissement du Conseil Régional de Lorraine qui fournit des espaces et services aux différentes ligues sportives régionales hébergées. Dans le cadre de la modernisation de son infrastructure réseau, la M2L souhaite mettre en place un pare-feu centralisé capable de filtrer le trafic entre les différents VLANs, de sécuriser les accès Internet et d'isoler les zones sensibles (DMZ, invités) du réseau interne.

Ce projet est réalisé dans le cadre de l'épreuve E6 du BTS SIO option SISR — réalisation professionnelle — à l'IPSSI Marne-La-Vallée.

Objectifs

  • VM OPNsense opérationnelle, interfaces WAN et LAN configurées
  • Routage inter-VLAN fonctionnel (VLANs 110 / 120 / 130 / 140 / 150)
  • Accès Internet fonctionnel depuis chaque VLAN
  • VLAN 140 invité isolé du réseau interne
  • NAT sortant opérationnel pour tous les VLANs

Architecture réseau

Le pare-feu OPNsense est déployé en VM sur un serveur Proxmox VE, avec deux interfaces : WAN (obtenue par DHCP sur le réseau école) et LAN trunk portant les sous-interfaces VLANs. Un switch L3 Cisco Catalyst assure le trunking 802.1Q vers OPNsense, et un switch L2 distribue les VLANs sur les ports d'accès. OPNsense centralise le routage inter-VLAN entre les cinq zones.

  • VLAN 110 — Admin : 10.20.110.0/24 — administration des équipements, passerelle 10.20.110.1
  • VLAN 120 — DMZ : 10.20.120.0/24 — serveur web exposé (10.20.120.10), passerelle 10.20.120.1
  • VLAN 130 — WiFi : 10.20.130.0/24 — clients sans-fil internes, passerelle 10.20.130.1
  • VLAN 140 — Invité : 10.20.140.0/24 — DHCP de 10.20.140.100 à 10.20.140.200, passerelle 10.20.140.1
  • VLAN 150 — Utilisateurs : 10.20.150.0/24 — postes de travail, passerelle 10.20.150.1

Fonctionnalités déployées

Routage inter-VLAN

5 sous-interfaces VLANs LAN avec passerelles dédiées, routage centralisé par OPNsense

Filtrage par règles

Admin/Utilisateurs/WiFi → WAN autorisé, Invité isolé du LAN, DMZ → LAN bloqué

Isolation VLAN invité

VLAN 140 cantonné à Internet, aucun flux vers LAN interne (ping bloqué)

NAT sortant

Masquerading sur tous les VLANs internes, accès Internet partagé via DHCP école

Redirection de port DMZ

Port forwarding 80/443 WAN → serveur web DMZ (10.20.120.10)

Interface d'administration

Accès web depuis VLAN Admin (10.20.110.1) — gestion centralisée navigateur

Réalisations techniques

  • Installation et configuration initiale d'OPNsense en VM sur Proxmox VE
  • Configuration de l'interface WAN en DHCP sur le réseau école
  • Création des cinq sous-interfaces VLANs côté LAN avec tagging 802.1Q
  • Définition du plan d'adressage 10.20.X.0/24 et passerelles par VLAN
  • Rédaction des règles de filtrage inter-VLAN avec politique de refus par défaut
  • Configuration du NAT sortant (masquerading) pour tous les VLANs internes
  • Mise en place d'une redirection de port WAN → DMZ pour le serveur web (80/443)
  • Configuration du trunk 802.1Q côté switches Cisco pour propager les VLANs

Tests de validation

  • Test d'accès Internet depuis chaque VLAN (110, 130, 140, 150)
  • Vérification du routage inter-VLAN par ping entre zones autorisées
  • Vérification de l'isolation du VLAN 140 invité (ping LAN bloqué)
  • Test d'accès au serveur web DMZ depuis le WAN via la redirection de port
  • Validation de l'isolation DMZ → LAN (flux bloqué)
